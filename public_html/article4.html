SOLID Object Oriented Design

"Let's talk Object-oriented design and TDD" by IMAZU Asami is licensed under CC BY-NC-SA 2.0

S stands for the single-responsibility principle. This means exactly that—that the class has only one responsibility. In our calculator, we show this principle by having separate classes for each of the mathematical operations.

O stands for the open-closed principle. This means that modification to the existing code cannot happen. This helps to prevent more bugs. However, bugs can be fixed in the already existing code. This is similar to the extension in calculation to add_calculation in our calculator. Instead of modifying existing code, we created a different object.

L stands for the Liskov Substitution Principle. This is when subclass of two classes is interchangeable between the superclass without consequences. Since our calculator does not have such similar classes, our code does not illustrate this principle.

I stands for the Interface Segregation Principle. This means that larger chunks of code are broken down into smaller more specific pieces of code. Since our calculator only demonstrates basic mathematical functions, it is already broken down into specific pieces of code. Therefore, this principle is not demonstrated in our code.

D stands for the Dependency Inversion Principle. This simply states that the higher modules do not depend on the lower modules. But that the lower modules depend on the higher ones. The calculator itself represents the higher modules, and the operations represent the lower modules.
