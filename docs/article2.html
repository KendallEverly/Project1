AAA Testing

When testing with Python, it is important to demonstrate the AAA pattern. AAA stands for Arrange-Act-Assert. The “Arrange” part of the test is where the objects are being set up to be tested. The “Act” part is the actual testing. The “Assert” part is where the outcome is included, such as a return value.

In the “Arrange” section, it shows the setup, data, created objects, and other similar arrangements. In the “Act” section, is shows the method being tested. In the “Assert” section, it verifies the test.

"""module calc"""

from calc.calculations.calculation import Calculation

class Addition(Calculation):

"""addition"""

def getresult(self):

"""get result addition"""

return self.value_a + self.value_b

The class would be the arrangement; the getresult is the act; and the return is the assert.

This type of testing pattern is fast and easy to understand. This means that it is best to not test a unit that branches. In fact, commenting which part of the test is the “Arrange”, “Act”, and “Assert” helps to differentiate each of the sections to make it easier to understand.
